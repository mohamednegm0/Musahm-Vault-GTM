## 8. Design Notes for Frontend Team

### Colors

| Token              | Value     | Usage                                     |
| ------------------ | --------- | ----------------------------------------- |
| `--brand-gold`     | `#c3924d` | Accents, CTAs, hover states               |
| `--primary-dark`   | `#1B4332` | Button backgrounds, section headers       |
| `--primary-white`  | `#FFFFFF` | Body background, button text              |
| `--text-primary`   | `#1A1A1A` | Body text                                 |
| `--text-secondary` | `#4A4A4A` | Subheadlines, descriptions                |
| `--bg-problem`     | `#F8F4F0` | Problem section background (warm neutral) |
| `--bg-solution`    | `#FFFFFF` | Solution section background               |
| `--bg-cta`         | `#1B4332` | Final CTA section background (dark)       |

### Layout

- **Hero:** Full-viewport height. Headline centered. CTA button prominent below subheadline. Subtle background pattern or gradient, no stock photos.
- **Problem section:** Warm neutral background to differentiate from hero. Three cards or stacked blocks, each with a pain-point question as the headline.
- **Solution section:** White background. Feature blocks in a 2x3 grid on desktop, stacked on mobile. Each block: icon (40px), headline (bold), 1-2 line description.
- **Social proof:** Client logos in a single row, centered, grayscale with hover-to-color effect.
- **How it works:** Three numbered steps in a horizontal timeline on desktop, vertical on mobile. Connecting line between steps.
- **FAQ:** Accordion pattern. Question visible, answer hidden until clicked.
- **Final CTA:** Dark background (`#1B4332`), white text, gold button (`#c3924d`).

### RTL Behavior

- Arabic is the primary layout. Set `dir="rtl"` on the `<html>` element for the Arabic version.
- All content sections flip horizontally in RTL: text right-aligned, icons/numbers on the right side.
- The "How It Works" timeline reads right-to-left in Arabic.
- FAQ accordion chevrons flip direction in RTL.
- Use CSS logical properties (`margin-inline-start`, `padding-inline-end`) instead of directional properties.
- Language toggle in the top navigation bar. Switching should not cause a page reload.

### Typography

- Headlines: Bold, high-contrast. Arabic headlines should use a Naskh-style or geometric Arabic web font (Noto Kufi Arabic or IBM Plex Arabic recommended).
- Body: 16px minimum on desktop, 14px minimum on mobile. Line height 1.6 for Arabic, 1.5 for English.
- Button text: 16px, semi-bold, uppercase English / standard Arabic.

### Responsiveness

- Breakpoints: 1200px (desktop), 768px (tablet), 480px (mobile).
- Feature grid: 3 columns desktop, 2 columns tablet, 1 column mobile.
- Hero CTA button: full-width on mobile, auto-width on desktop.
- FAQ section: full-width on all breakpoints.

### Interaction

- CTA buttons: hover state with slight scale (1.02) and shadow. No bounce animations.
- Scroll-triggered fade-in for feature blocks and steps (subtle, 200ms).
- FAQ accordion: smooth height transition (300ms ease).
- No auto-playing videos or carousels.

### Accessibility

- All interactive elements keyboard-accessible.
- CTA buttons: minimum contrast ratio 4.5:1.
- FAQ: `aria-expanded` attributes on accordion triggers.
- Arabic and English versions: `lang` attribute set correctly on all text blocks.
- Skip-to-content link for screen readers.
