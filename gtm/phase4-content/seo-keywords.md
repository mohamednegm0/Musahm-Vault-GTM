### Notes on Arabic Search Volume Data

> **Honest limitation:** Arabic keyword search volume data is unreliable. Google Keyword Planner, Ubersuggest, and Ahrefs all undercount Arabic queries because:
>
> - Arabic morphology creates variants (حوكمة، الحوكمة، محوكم) that tools treat as separate keywords
> - Regional Arabic variations (Gulf vs. Levantine vs. Egyptian) fragment volume
> - Many Saudi business decision-makers search in English even for Arabic-market products
>
> **Recommendation:** Use Google Trends (set to Saudi Arabia) to validate relative demand between Arabic terms. Use Google Search Console post-launch to discover actual query patterns. Do not over-invest in "volume estimates" for Arabic keywords.

## 5. Technical SEO Checklist

### Schema Markup Types

| Page         | Schema Type                | Key Properties                                                                      |
| ------------ | -------------------------- | ----------------------------------------------------------------------------------- |
| Homepage     | `Organization` + `WebSite` | name, url, logo, sameAs (social profiles)                                           |
| Product Page | `SoftwareApplication`      | name, applicationCategory ("BusinessApplication"), operatingSystem ("Web"), offers  |
| Landing Page | `Product` + `WebPage`      | name, description, brand, offers (beta signup)                                      |
| Blog Posts   | `Article`                  | headline, alternativeHeadline (AR/EN), datePublished, author, publisher, inLanguage |
| FAQ Page     | `FAQPage` + `Question`     | name (question), acceptedAnswer (answer text)                                       |
| Pricing Page | `Product` + `Offer`        | price, priceCurrency (SAR), availability                                            |

### Hreflang Implementation

```html
<!-- On every page, include both language versions -->
<!-- Arabic page -->
<link
  rel="alternate"
  hreflang="ar"
  href="https://vault.musahm.com/ar/page-slug"
/>
<link
  rel="alternate"
  hreflang="en"
  href="https://vault.musahm.com/en/page-slug"
/>
<link
  rel="alternate"
  hreflang="x-default"
  href="https://vault.musahm.com/ar/page-slug"
/>
```

**Notes:**

- Arabic is `x-default` because it is the primary language and the primary market language
- Use `ar` not `ar-SA` unless you plan separate Arabic variants for other countries
- Every Arabic page must point to its English equivalent and vice versa
- Hreflang must be reciprocal (both pages point to each other)

### On-Page SEO Checklist

- [ ] Title tags: `{Page Title} | مساهم Vault` (AR) / `{Page Title} | Musahm Vault` (EN)
- [ ] Title tag length: 50-60 characters
- [ ] Meta descriptions: unique per page, 120-155 characters, include primary keyword
- [ ] H1: one per page, contains primary keyword, matches (but does not duplicate) title tag
- [ ] H2/H3: use secondary keywords naturally in subheadings
- [ ] Image alt text: descriptive, bilingual where applicable
- [ ] Internal links: every blog post links to at least 2 other pages
- [ ] Canonical tags: self-referencing on each page
- [ ] Open Graph tags: og:title, og:description, og:image, og:locale (ar_SA / en_US)
- [ ] Twitter Card tags: twitter:card (summary_large_image), twitter:title, twitter:description
- [ ] URL structure: lowercase, hyphens, language prefix (/ar/ or /en/)
- [ ] No orphan pages: every page reachable from at least one internal link

### Core Web Vitals Targets

| Metric                         | Target  | Note                                                 |
| ------------------------------ | ------- | ---------------------------------------------------- |
| LCP (Largest Contentful Paint) | < 2.5s  | Critical for landing page hero section               |
| FID (First Input Delay)        | < 100ms | Keep JS payload minimal                              |
| CLS (Cumulative Layout Shift)  | < 0.1   | Arabic web fonts load can cause CLS -- preload fonts |
| TTFB (Time to First Byte)      | < 600ms | Cloudflare CDN should handle this                    |

### Arabic-Specific SEO Considerations

- [ ] Arabic content is real content, not machine-translated (Google can detect translated content)
- [ ] Arabic meta descriptions are written natively, not translated from English
- [ ] Arabic URLs use transliterated slugs (/ar/blog/hawkama-watha2iq) or English slugs (/ar/blog/document-governance) -- both are valid, but be consistent
- [ ] Right-to-left text rendering does not break structured data
- [ ] Arabic content includes natural keyword variants (حوكمة، الحوكمة، محوكم، حوكمي)

## 6. Google Search Console Setup Instructions

### Step 1: Verify Domain Ownership

1. Go to [Google Search Console](https://search.google.com/search-console)
2. Click "Add property"
3. Select "Domain" and enter `musahm.com` (covers all subdomains including vault.musahm.com)
4. Verify via DNS TXT record:
   - Log into your DNS provider (likely Cloudflare based on deployment stack)
   - Add a TXT record with the verification string Google provides
   - Wait for propagation (can take up to 48 hours, usually minutes on Cloudflare)
5. Also add `vault.musahm.com` as a separate URL-prefix property for granular data

### Step 2: Submit Sitemaps

1. In Search Console, go to "Sitemaps" in the left sidebar
2. Submit:
   - `https://vault.musahm.com/sitemap.xml` (sitemap index)
   - `https://vault.musahm.com/sitemap-ar.xml` (if separate)
   - `https://vault.musahm.com/sitemap-en.xml` (if separate)
3. Verify all sitemaps show status "Success"

### Step 3: Configure Settings

1. **Country targeting:** Go to Settings > International Targeting > Country tab > Select "Saudi Arabia"
2. **Language:** Handled via hreflang tags (configured in Step 5 of Technical SEO above)
3. **Preferred domain:** Ensure `https://vault.musahm.com` (with https, with or without www -- pick one and redirect the other)

### Step 4: Set Up Performance Monitoring

1. **Link to Google Analytics 4:** In Search Console > Settings > Associations > Associate with GA4 property
2. **Create custom views for tracking:**
   - Filter by language (AR vs EN) to compare performance
   - Filter by query to track target keywords
   - Filter by page to track individual blog post performance

### Step 5: Keyword Tracking Plan

After 2-4 weeks of data collection, review:

| Metric                 | What to Look For                                 | Action                                                            |
| ---------------------- | ------------------------------------------------ | ----------------------------------------------------------------- |
| Impressions by query   | Which Arabic queries are generating impressions  | Validate against target keyword list. Add new discoveries.        |
| CTR by query           | Which queries have high impressions but low CTR  | Rewrite title tags and meta descriptions for those queries        |
| Position by query      | Which target keywords are ranking positions 5-20 | These are "striking distance" keywords. Optimize the target page. |
| Clicks by page         | Which pages drive the most organic traffic       | Double down with internal links and content updates               |
| Impressions by country | Traffic from Saudi Arabia vs. other countries    | Confirm Saudi Arabia is the dominant source                       |
| Coverage report        | Indexing errors, excluded pages                  | Fix any crawl errors, especially for Arabic pages                 |

### Step 6: Search Console Alerts to Configure

- **Index coverage issues:** Email alerts for any new errors
- **Manual actions:** Email alerts (unlikely but important)
- **Security issues:** Email alerts
- **Core Web Vitals:** Monitor monthly
