## Slide 7: Feature Deep-Dive: Access Control + Audit Trail

**Slide Title:** تحكم كامل. أثر كامل. / Complete Control. Complete Trail.

**Content:**

- **خمسة مستويات صلاحيات:** عارض، معلّق، محرر، منظّم، مدير
  Five access roles: Viewer, Commenter, Editor, Organizer, Admin.
- **صلاحيات على مستوى الوثيقة:** كل وثيقة بتحكم مستقل
  Per-document ACLs: Every document has independent access control.
- **مشاركة خارجية بكلمة مرور لمرة واحدة (OTP)** مع علامة مائية
  OTP-verified external sharing with watermarked copies.
- **سجل تدقيق شامل:** كل عملية مسجّلة بالمستخدم والتاريخ والتفاصيل
  Full audit trail: Every action logged with user identity, timestamp, and details.

**Speaker Notes:**

Access control in Vault operates on five hierarchical levels. A Viewer can read documents but not modify them. A Commenter can annotate. An Editor can make changes. An Organizer governs a specific workspace. And an Admin has full platform control. Beyond roles, every individual document can have its own access permissions. If a contract should only be visible to the CFO and Legal Counsel, you set that at the document level. When you need to share a document externally, with an auditor, a legal advisor, a regulator, Vault generates an OTP-verified link. The recipient gets a watermarked copy. You see who opened it, when, and from where. And every action across the entire platform feeds into a complete audit trail. User, timestamp, action details. No one deletes without a record. No one accesses without a trace.

**Visual Direction:**

Two-panel layout. Left panel: role hierarchy diagram showing the five levels as a stepped pyramid. Right panel: audit trail visualization showing a timeline of actions with user avatars, timestamps, and action types. Use the brand gold (#c3924d) for access-level highlighting.

**Key Persona:** CFO (audit trail for compliance), Legal Counsel (access control for confidentiality).

## Slide 8: Feature Deep-Dive: Workflows + Version Control

**Slide Title:** لا وثيقة تُنشر بدون موافقة. / No Document Goes Live Without Approval.

**Content:**

- **مصمم سير عمل مرئي:** ارسم سلسلة الموافقة والمراجعة بسحب وإفلات
  Visual workflow designer: Build approval and review chains with drag-and-drop.
- **سير عمل مانع:** لا يمكن نشر الوثيقة حتى اكتمال جميع الموافقات
  Blocking workflows: Documents cannot be published until all approvals are complete.
- **التحكم بالإصدارات:** كل تعديل يُحفظ كإصدار منفصل
  Version control: Every edit saved as a separate version.
- **السحب والإرجاع (Check-in/Check-out):** يمنع التعديل المتزامن ويحفظ سلسلة التعديلات
  Check-in/check-out: Prevents simultaneous editing and preserves the edit chain.

**Speaker Notes:**

One of the most dangerous things in governance is a document that goes live without the right approvals. In Vault, that cannot happen. The workflow engine lets you design approval chains visually. You drag, you drop, you define who needs to approve, in what order, and what happens if they reject. These workflows are blocking, which means the system enforces them. If the CFO has not signed off on a financial policy, it stays in draft. Period. On the version control side, every edit creates a new version. You can see who changed what, when, and roll back if needed. And the check-in/check-out system prevents two people from editing the same document simultaneously, which eliminates version conflicts entirely.

**Visual Direction:**

Main visual: a simplified workflow diagram showing a document moving through stages (Draft, Review, Approval, Published) with role icons at each gate. Secondary element: version history timeline showing multiple versions of a document with timestamps and user identifiers. Clean lines, no clutter.

**Key Persona:** Board Secretary (daily workflow management), Legal Counsel (approval enforcement).
