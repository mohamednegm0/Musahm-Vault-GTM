## ARCHITECTURE GRAPH

### Layer Dependencies
```
Frontend/apps/web → [apiClient.ts] → Backend/Vault/API/Controllers
Backend/Vault/API → Backend/Vault/Service → Backend/Vault/Repository → MongoDB
Backend/Vault/Core ← (shared by API, Service, Repository)
Backend/Vault/MCP-Bots → Backend/Vault/API (HTTP)
```

### Backend Clean Architecture
```
API/Controllers/{X}Controller.cs
  → uses I{X}Service (from Core/Interfaces/Service/)
  → impl {X}Service (from Service/Services/)
    → uses IBaseRepository<{Entity}> (from Core/Interfaces/Repository/)
    → impl BaseRepository<{Entity}> (from Repository/Repositories/)
      → MongoDB IMongoCollection<{Entity}>
```

### Frontend Data Flow
```
Page.tsx → src/api/{module}.ts → apiClient.ts → axios → Backend API
Page.tsx → useAuth() [AuthContext] → JWT decode → user state
Page.tsx → useWorkspace() [WorkspaceContext] → selected workspace
Page.tsx → useLanguage() [LanguageContext] → ar/en toggle
```

## KEY ARCHITECTURAL PATTERNS

### Blocking Workflows
Document upload/update/delete/share → check WorkflowAssignment for applicable workflow
If workflow found → Document status set to "Pending" → WorkflowInstance created → Tasks generated per step
When workflow completes → Document status set to "Active" (for share: invitation emails sent)
HTTP 460 = WorkflowRequired (custom status code)

### Dual File Storage
DocumentVersion.FileContent (byte[]) → inline binary in MongoDB document
GridFS (fs.files + fs.chunks) → used for large files
Service tries inline first, falls back to GridFS for download

### OTP External Access
POST /api/Invitations/generate-otp [AllowAnonymous] → sends 6-digit OTP to email
POST /api/Invitations/verify-otp [AllowAnonymous] → validates OTP (10-min expiry), returns watermarked file stream
Invitation stores watermarked copy (WatermarkHelper stamps recipient email)

### Identity Resolution (per request)
BaseApiController (IAsyncActionFilter) → parses ClaimsIdentity → CurrentUser DTO → loads Permissions from MongoDB
TenantId derived from JWT tenant_id claim, falls back to CompanyId
GRC Service forwards Bearer token as x-api-key header to external GRC API
